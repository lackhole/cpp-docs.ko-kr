# <a name="sequence-of-operations-for-building-mfc-applications"></a>MFC 애플리케이션을 빌드하는 작업 시퀀스

다음 표에서는 MFC 응용 프로그램을 개발할 때 일반적으로 수행할 수 있는 일반적인 시퀀스에 대해 설명 합니다.

### <a name="sequence-for-building-an-application-with-the-framework"></a>프레임 워크를 사용 하 여 응용 프로그램을 빌드하기 위한 시퀀스

|태스크|너 해|프레임 워크는|
|----------|------------|------------------------|
|기본 응용 프로그램을 만듭니다.|[MFC 응용 프로그램 마법사](../mfc/reference/mfc-application-wizard.md)를 실행 합니다. 옵션 페이지에서 원하는 옵션을 지정 합니다. 응용 프로그램을 COM 구성 요소, 컨테이너 또는 둘 다로 만드는 옵션이 있습니다. 자동화 추가 응용 프로그램을 데이터베이스에서 인식 하도록 합니다.|MFC 응용 프로그램 마법사는 응용 프로그램, 문서, 뷰 및 프레임 창에 대 한 소스 파일을 포함 하 여 기본 응용 프로그램용 파일을 만듭니다. 리소스 파일입니다. 프로젝트 파일 그리고 다른 사용자가 사양에 맞게 조정 되었습니다.|
|사용자 고유의 코드 줄을 추가 하지 않고 프레임 워크 및 MFC 응용 프로그램 마법사에서 제공 하는 작업을 확인 합니다.|기본 응용 프로그램을 빌드하고 시각적 C++개체에서 실행 합니다.|실행 중인 기본 응용 프로그램은 프레임 워크에서 많은 표준 **파일**, **편집**, **보기**및 **도움말** 메뉴 명령을 파생 시킵니다. MDI 응용 프로그램의 경우 완벽 하 게 작동 하는 Windows 메뉴가 표시 되 고 프레임 워크는 MDI 자식 창의 생성, 정렬 및 소멸을 관리 합니다.|
|응용 프로그램의 사용자 인터페이스를 생성 합니다.|시각적 C++ [리소스 편집기](../windows/resource-editors.md) 를 사용 하 여 응용 프로그램의 사용자 인터페이스를 시각적으로 편집 합니다.<br /><br /> -메뉴를 만듭니다.<br />-가속기를 정의 합니다.<br />-대화 상자를 만듭니다.<br />-비트맵, 아이콘 및 커서를 만들고 편집 합니다.<br />-MFC 응용 프로그램 마법사에서 만든 도구 모음을 편집 합니다.<br />-기타 리소스를 만들고 편집 합니다.<br /><br /> 대화 상자 편집기에서 대화 상자를 테스트할 수도 있습니다.|MFC 응용 프로그램 마법사에서 만든 기본 리소스 파일은 필요한 많은 리소스를 제공 합니다. 시각적 C++ 개체를 사용 하면 기존 리소스를 편집 하 고 쉽고 시각적으로 새 리소스를 추가할 수 있습니다.|
|메뉴를 처리기 함수에 매핑합니다.|**클래스 뷰** (또는 [클래스 마법사](reference/mfc-class-wizard.md)의 **명령** 탭)의 [ **속성** 창](/visualstudio/ide/reference/properties-window) 에서 **이벤트** 단추를 사용 하 여 메뉴 및 액셀러레이터를 코드의 처리기 함수에 연결할 수 있습니다.|이러한 도구는 메시지 맵 항목과 빈 함수 템플릿을 지정 하는 소스 파일에 삽입 하 고 여러 수동 코딩 작업을 관리 합니다.|
|처리기 코드를 작성 합니다.|클래스 뷰를 사용 하 여 소스 코드 편집기에서 코드로 직접 이동 합니다. 처리기 함수에 대 한 코드를 입력 합니다. 클래스 뷰 사용에 대 한 자세한 내용과 프로젝트에 코드를 추가 하는 마법사에 대 한 자세한 내용은 [코드 마법사로 기능 추가](../ide/adding-functionality-with-code-wizards-cpp.md)를 참조 하세요.|클래스 뷰 편집기를 열고 빈 함수 템플릿으로 스크롤하고 커서를 위치에 배치 합니다.|
|도구 모음 단추를 명령에 매핑합니다.|단추를 적절 한 명령 ID에 할당 하 여 도구 모음에 있는 각 단추를 메뉴 또는 액셀러레이터 명령에 매핑합니다.|프레임 워크는 도구 모음 단추의 그리기, 활성화, 비활성화, 확인 및 기타 시각적 요소를 제어 합니다.|
|처리기 함수를 테스트 합니다.|프로그램을 다시 빌드하고 기본 제공 디버깅 도구를 사용 하 여 처리기가 제대로 작동 하는지 테스트 합니다.|코드를 단계별로 실행 하거나 추적 하 여 처리기가 호출 되는 방법을 확인할 수 있습니다. 처리기 코드를 작성 한 경우 처리기는 명령을 실행 합니다. 프레임 워크는 처리 되지 않는 메뉴 항목 및 도구 모음 단추를 자동으로 사용 하지 않도록 설정 합니다.|
|[대화 상자](../mfc/dialog-boxes.md)를 추가 합니다.|대화 상자 편집기를 사용 하 여 대화 상자 템플릿 리소스를 디자인 합니다. 그런 다음 대화 상자 클래스와 대화 상자를 처리 하는 코드를 만듭니다.|프레임 워크는 대화 상자를 관리 하 고 사용자가 입력 한 정보를 쉽게 검색할 때 사용 됩니다.|
|대화 상자 데이터를 초기화 하 고, 유효성을 검사 하 고, 검색 합니다.|또한 대화 상자의 컨트롤을 초기화 하 고 유효성을 검사 하는 방법을 정의할 수 있습니다. Visual Studio를 사용 하 여 대화 상자 클래스에 멤버 변수를 추가 하 고 대화 상자 컨트롤에 매핑합니다. 사용자가 데이터를 입력할 때 각 컨트롤에 적용할 유효성 검사 규칙을 지정 합니다. 원하는 경우 고유한 사용자 지정 유효성 검사를 제공 합니다.|프레임 워크는 대화 상자 초기화 및 유효성 검사를 관리 합니다. 사용자가 잘못 된 정보를 입력 하면 프레임 워크에서 메시지 상자를 표시 하 고 사용자가 데이터를 다시 입력할 수 있도록 합니다.|
|추가 클래스를 만듭니다.|클래스 뷰 사용 하 여 MFC 응용 프로그램 마법사에서 자동으로 만든 문서, 뷰 및 프레임 창 클래스 이외의 클래스를 만들 수 있습니다. 추가 데이터베이스 레코드 집합 클래스, 대화 상자 클래스 등을 만들 수 있습니다. 클래스 뷰를 사용 하 여 MFC 클래스에서 파생 되지 않은 클래스를 만들 수 있습니다.|클래스 뷰는 이러한 클래스를 소스 파일에 추가 하 고 처리 하는 모든 명령에 대 한 연결을 정의 하는 데 도움을 줍니다.|
|즉시 사용 가능한 구성 요소를 응용 프로그램에 추가 합니다.|를 사용 `New Item dialog box` 하 여 다양 한 항목을 추가할 수 있습니다.|이러한 항목을 응용 프로그램에 쉽게 통합 하 여 많은 작업을 수행할 수 있습니다.|
|문서 클래스를 구현 합니다.|응용 프로그램별 문서 클래스를 구현 합니다. 데이터 구조를 보유할 멤버 변수를 추가 합니다. 멤버 함수를 추가 하 여 데이터에 대 한 인터페이스를 제공 합니다.|프레임 워크는 문서 데이터 파일과 상호 작용 하는 방법을 이미 알고 있습니다. 문서 파일을 열거나 닫고, 문서의 데이터를 읽고 쓰고, 다른 사용자 인터페이스를 처리할 수 있습니다. 문서의 데이터를 조작 하는 방법에 집중할 수 있습니다.|
|열기, 저장 및 다른 이름으로 저장 명령을 구현 합니다.|문서의 `Serialize` 멤버 함수에 대 한 코드를 작성 합니다.|프레임 워크는 **파일** 메뉴의 **열기**, **저장**및 **다른 이름으로 저장** 명령에 대 한 대화 상자를 표시 합니다. `Serialize` 멤버 함수에 지정 된 데이터 형식을 사용 하 여 문서를 쓰고 다시 읽습니다.|
|뷰 클래스를 구현 합니다.|문서에 해당 하는 뷰 클래스를 하나 이상 구현 합니다. 클래스 뷰를 사용 하 여 사용자 인터페이스에 매핑한 뷰의 멤버 함수를 구현 합니다. [CListView](../mfc/reference/clistview-class.md) 및 [ctreeview](../mfc/reference/ctreeview-class.md)를 비롯 한 다양 한 [CView](../mfc/reference/cview-class.md)파생 클래스를 사용할 수 있습니다.|프레임 워크는 문서와 뷰 간의 관계를 대부분 관리 합니다. 뷰의 멤버 함수는 보기의 문서에 액세스 하 여 화면 또는 인쇄 된 페이지에서 이미지를 렌더링 하 고 사용자 편집 명령에 대 한 응답으로 문서의 데이터 구조를 업데이트 합니다.|
|기본 인쇄를 향상 시킵니다.|다중 페이지 인쇄를 지원 해야 하는 경우 뷰 멤버 함수를 재정의 합니다.|프레임 워크는 **파일** 메뉴의 **인쇄**, **페이지 설정**및 **인쇄 미리 보기** 명령을 지원 합니다. 문서를 여러 페이지로 분리 하는 방법을 알려 주어 야 합니다.|
|스크롤을 추가 합니다.|스크롤을 지원 해야 하는 경우 [CScrollView](../mfc/reference/cscrollview-class.md)에서 뷰 클래스 또는 클래스를 파생 시킵니다.|뷰 창이 너무 작으면 뷰를 통해 스크롤 막대가 자동으로 추가 됩니다.|
|폼 보기를 만듭니다.|대화 상자 템플릿 리소스에 대 한 보기를 만들려면 [CFormView](../mfc/reference/cformview-class.md)에서 뷰 클래스 또는 클래스를 파생 시킵니다.|뷰는 대화 상자 템플릿 리소스를 사용 하 여 컨트롤을 표시 합니다. 사용자는 컨트롤에서 컨트롤로 tab 키를 누를 수 있습니다.|
|데이터베이스 폼을 만듭니다.|양식 기반 데이터 액세스 응용 프로그램을 사용 하려면 [CRecordView](../mfc/reference/crecordview-class.md) (ODBC 프로그래밍의 경우)에서 뷰 클래스를 파생 시킵니다.|뷰는 폼 보기와 같이 작동 하지만 해당 컨트롤은 데이터베이스 테이블을 나타내는 [CRecordset](../mfc/reference/crecordset-class.md) 개체의 필드에 연결 됩니다. MFC는 컨트롤과 레코드 집합 간에 데이터를 이동 합니다.|
|간단한 텍스트 편집기를 만듭니다.|뷰가 간단한 텍스트 편집기가 되도록 하려면 [Ceditview](../mfc/reference/ceditview-class.md) 또는 [CRichEditView](../mfc/reference/cricheditview-class.md)에서 뷰 클래스 또는 클래스를 파생 시킵니다.|뷰는 편집 함수, 클립보드 지원 및 파일 입/출력을 제공 합니다. `CRichEditView`스타일 텍스트를 제공 합니다.|
|분할자 창을 추가 합니다.|창 분할을 지원 하려면 SDI 프레임 창이 나 MDI 자식 창에 [CSplitterWnd](../mfc/reference/csplitterwnd-class.md) 개체를 추가 하 고 창의 [OnCreateClient](../mfc/reference/cframewnd-class.md#oncreateclient) 멤버 함수에 후크 합니다.|프레임 워크는 스크롤 막대 옆에 분할자 상자 컨트롤을 제공 하 고 뷰를 여러 창으로 분할 하는 것을 관리 합니다. 사용자가 창을 분할 하면 프레임 워크에서 추가 뷰 개체를 만들어 문서에 연결 합니다.|
|응용 프로그램을 빌드, 테스트 및 디버그 합니다.|시각적 개체 C++ 기능을 사용 하 여 응용 프로그램을 빌드, 테스트 및 디버그할 수 있습니다.|시각적 C++ 개체를 사용 하면 컴파일, 링크 및 기타 옵션을 조정할 수 있습니다. 또한 소스 코드와 클래스 구조를 찾아볼 수 있습니다.|
